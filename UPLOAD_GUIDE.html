GitHub — Criar repositório e fazer upload
Upload manual de todos os arquivos respeitando a estrutura de pastas
⚠️ Como o GitHub funciona para upload manual: O GitHub não cria pastas automaticamente. Para criar a estrutura de pastas, você precisa incluir o caminho completo no nome do arquivo ao fazer upload. Ex: ao criar um arquivo em src/app/, você digita src/app/page.tsx no campo de nome.
1

Criar o repositório

Acesse github.com → clique em New repository (botão verde "+" no canto superior direito) → Nome: mbx-lms → Public → Create repository

2

Método de upload recomendado

Na página do repositório vazio, clique em "creating a new file". Para cada arquivo, você vai:

① Digitar o caminho completo no campo de nome (ex: src/app/page.tsx)
② Colar o conteúdo do arquivo na área de texto
③ Clicar em Commit changes

💡 Dica: Para criar pastas, basta incluir o caminho no nome do arquivo. Ao digitar src/app/page.tsx e pressionar /, o GitHub cria automaticamente as pastas src/ e src/app/.
Ordem de upload — faça nessa sequência:

Vercel — Deploy automático pelo GitHub
Conecta o repositório e configura as variáveis de ambiente
1

Importar projeto no Vercel

Acesse vercel.com → faça login (recomendado: "Continue with GitHub") → clique em Add New Project → selecione o repositório mbx-lms → clique em Import

2

Framework e configuração

O Vercel detecta automaticamente o Next.js. Não mude nada em "Build and Output Settings" — o vercel.json já configura tudo.

3

Supabase — Configurar URLs de Auth
Necessário para o login funcionar em produção
1

Abra as configurações de Auth

No Supabase, vá em Authentication (menu lateral) → URL Configuration

2

Configure Site URL e Redirect URLs

Copie a URL gerada pelo Vercel e configure:

# Site URL (URL principal do seu projeto no Vercel)
Site URL: https://mbx-lms-SEU-HASH.vercel.app

# Redirect URLs (adicionar as duas)
Redirect URLs:
  https://mbx-lms-SEU-HASH.vercel.app/**
  http://localhost:3000/** (para dev local)

Clique em Save.

3

Atualizar NEXT_PUBLIC_SITE_URL no Vercel

No Vercel, vá em Project Settings → Environment Variables → edite NEXT_PUBLIC_SITE_URL com a URL real → Save → vá em Deployments → clique em Redeploy no último deploy.

Confirmar no Supabase
